## 1. ภาพรวมระบบ

### PEA Brain คืออะไร?

**PEA Brain** คือระบบ AI อัจฉริยะที่ช่วย **กองจัดหาพัสดุ** ของ PEA ในการ:

| ความสามารถ | รายละเอียด |
|---|---|
| 🔍 **เฝ้าระวังความเสี่ยง** | ตรวจจับสถานการณ์พัสดุวิกฤตแบบ Real-time |
| 🧠 **วิเคราะห์ด้วย AI** | Amazon Nova Pro วิเคราะห์ข้อมูลจริงจากฐานข้อมูล |
| 📊 **แนะนำแผนจัดซื้อ** | เปรียบเทียบ Plan A vs Plan B พร้อมคำนวณ TCO |
| 📦 **ติดตามการจัดส่ง** | เฝ้าดูสถานะแผนที่อนุมัติไปแล้ว |
| 💬 **ถาม-ตอบ AI** | คุยกับ AI Copilot ถามข้อมูลเชิงลึกได้ทันที |

## 3. หน้าหลัก: Overview Dashboard

### 3.1 Hero Banner (ส่วนบนสุด)

แถบ Gradient สีม่วง-ทอง แสดงข้อมูลสรุปสำคัญ:

|---|---|---|

### 3.2 KPI Cards (5 การ์ด)

แถว KPI แสดงตัวเลขสำคัญ 5 ช่อง:

1. **📦 พัสดุ Critical** — จำนวนพัสดุที่สต็อกต่ำกว่าเกณฑ์อันตราย
2. **⚠️ แจ้งเตือนทั้งหมด** — จำนวนแจ้งเตือนความเสี่ยงจาก AI
3. **🕐 Lead Time เฉลี่ย** — ระยะเวลาเฉลี่ยในการจัดส่งพัสดุ (สัปดาห์)
4. **💰 มูลค่าความเสี่ยง** — ผลรวมมูลค่า VaR ของทุกรายการ
5. **✅ แผนอนุมัติแล้ว** — จำนวนแผนจัดซื้อที่ผ่านการอนุมัติ

### 3.3 ปุ่มสำคัญ

### 3.4 ตารางแจ้งเตือนด่วน

ด้านล่าง KPI จะแสดงตารางแจ้งเตือนแบบย่อ ซึ่งสามารถ:

## 4. หน้า Risk Management

### 4.1 แถบกรอง (Filter Bar)

| ตัวกรอง | ตัวเลือก | หน้าที่ |
|---|---|---|
| ระดับความเสี่ยง | ทั้งหมด / 🔴 Critical / 🟡 Warning / 🔵 Info | กรองตามระดับ |
| ค้นหา | พิมพ์ชื่อ/รหัสพัสดุ | ค้นหาพัสดุเฉพาะ |

### 4.2 ตาราง Risk Alerts

ตารางหลักแสดงข้อมูลทุกรายการ:

| คอลัมน์ | คำอธิบาย |
|---|---|
| ลำดับ | ลำดับความสำคัญ (วงกลมแดง = Critical) |
| รายการแจ้งเตือน | รหัส SAP + ชื่อพัสดุ + หมวดหมู่ |
| ระดับความเสี่ยง | Badge แสดงระดับ Critical/Warning/Info |
| Stock / Safety | สต็อกปัจจุบัน / Safety Stock (มีแถบ % สี) |
| Lead Time | ระยะเวลาจัดส่ง (สัปดาห์) |
| มูลค่าความเสี่ยง (VaR) | มูลค่าผลกระทบหากเกิดปัญหา |
| แนวโน้ม | กราฟ Sparkline 12 เดือน |
| การดำเนินการ | ปุ่มดำเนินการ (ดูรายละเอียดด้านล่าง) |

### 4.3 ปุ่มดำเนินการในตาราง

| สถานะ | ปุ่ม | การทำงาน |
|---|---|---|
| ยังไม่มีแผน | 🔴 **ให้ AI เข้าไปวิเคราะห์ →** | เปิดหน้า AI วิเคราะห์เต็มจอ |
| มีแผนแล้ว | ✅ **มีแผนแล้ว ▼** | กดขยายดูรายละเอียดแผนที่อนุมัติ |

### 4.4 รายการ Critical — คำแนะนำ AI

รายการ **Critical** ที่ยังไม่มีแผน จะมีแถวเพิ่มเติมแสดง:
```
⭐ คำแนะนำ AI: สต็อกเหลือ 12 เครื่อง ต้องเร่งจัดซื้อ...  ดูเหตุผลและการคำนวณ →
```
กดที่ **"ดูเหตุผลและการคำนวณ →"** เพื่อเปิดหน้า AI วิเคราะห์

### 4.5 ส่วนขยาย (Expand) — แผนที่อนุมัติ

เมื่อกดขยายรายการที่มีแผนแล้ว จะแสดง:

1. **Executive Summary** — สรุปสำหรับผู้บริหาร
2. **ตาราง AI Insights** — Demand / Market / Supplier Analysis
3. **ตาราง Financial & Procurement Summary** — เปรียบเทียบ TCO
4. **แผนที่ AI แนะนำ (อนุมัติแล้ว)** — สรุปแผนพร้อมรายละเอียดครบ

## 6. หน้า Tracking & Monitoring

### 6.1 สถิติสรุปด้านบน (4 การ์ด)

| การ์ด | ข้อมูล |
|---|---|
| 📦 กำลังติดตาม | จำนวนพัสดุที่มีแผนดำเนินการ |
| ✅ ดำเนินการเสร็จ | จำนวนที่ส่งมอบครบถ้วน |
| ⚠️ ต้องดำเนินการ | จำนวนที่รอการจัดการ |
| 💰 มูลค่ารวม | มูลค่ารวมทุกแผนที่อนุมัติ |

### 6.2 รายการพัสดุ

แต่ละรายการแสดง:
- **ชื่อพัสดุ + รหัส** — เช่น "หม้อแปลง 160 kVA (10067)"
- **สต็อกคงเหลือ** — "12 / 200 = ใช้ได้อีก 1 วัน"
- **สถานะ Stepper** — 4 ขั้นตอน:

```
[1. วิเคราะห์] → [2. อนุมัติแผน] → [3. จัดซื้อ (PO)] → [4. ส่งมอบ]
     ✅               ✅                🔵 กำลังดำเนินการ      ⬜ รอ
```

### 6.3 ส่วนขยาย (กดที่รายการ)

**กรณีที่ยังไม่มีแผน:**
- แสดงกล่อง **"AI เฝ้าระวัง"** พร้อมข้อความเตือน
- ปุ่ม **"AI วิเคราะห์แผนรับมือทันที"** → เรียก AI วิเคราะห์

**กรณีที่มีแผนแล้ว:**

#### A) สรุปแผนดำเนินการ (Approved Plan Summary)

ตาราง 2 แถว:
- **แผนหลักเดิม** — แผนรอส่งมอบปกติ
- **แผนรับมือฉุกเฉิน** (ถ้ามี) — แผนที่ AI แนะนำเพิ่มเติม

#### B) AI เฝ้าระวังการจัดส่ง (Tracking)

กล่องสีฟ้าแสดง:
```
🚚 AI เฝ้าระวังการจัดส่ง (Tracking)
พบความเสี่ยง: ผู้ผลิตหลักแจ้งปัญหา Supply Chain อาจล่าช้า 15 วัน...
```

ปุ่ม **"ให้ AI วิเคราะห์แผนสำรอง"** → เรียก AI สร้างแผนฉุกเฉิน

#### C) Deep Analysis (ผลวิเคราะห์เชิงลึก)

เมื่อ AI วิเคราะห์เสร็จ จะแสดง:

1. **Multi-Agent Analysis Table:**
   - 🎯 Demand Planner AI
   - 💼 Procurement AI
   - 📦 Warehouse AI
   - ⚠️ Impact Analysis
   - ⏱️ Speed Comparison
   - 💰 Financial Impact

2. **Executive Summary** — สรุปสั้นๆ

3. **ตารางเปรียบเทียบแผนฉุกเฉิน (Emergency Decision Matrix):**
   - ✅ AI แนะนำ ทางเลือกที่ 1 vs ทางเลือกที่ 2
   - เหตุผล, รายละเอียด, ผลลัพธ์, ความเสี่ยง, ขั้นตอนดำเนินการ

4. **ปุ่มเลือกแผน** — กด "อนุมัติทางเลือกที่ 1" หรือ "อนุมัติทางเลือกที่ 2"

### 6.4 ปุ่มสร้างใบสั่งซื้อ (PO)

ใน Stepper ขั้นที่ 3 จะมีปุ่ม **"📄 สร้าง PO อัตโนมัติ"**:
1. กดปุ่ม → ระบบจำลองการสร้างใบสั่งซื้อ
2. แสดง Progress: ตรวจสอบคงคลัง → ส่งข้อมูลไป SAP → เสร็จสมบูรณ์
3. สร้างเอกสาร **ใบสั่งซื้อ (Purchase Order)** ที่มี:
   - หัวเอกสาร PEA + เลข PO
   - ข้อมูลผู้ขาย (Vendor)
   - ตารางรายการสั่งซื้อ + ราคา + VAT
   - ลายเซ็นอนุมัติอัตโนมัติ
4. สามารถ **พิมพ์เอกสาร** ได้

### 7.1 Project Roadmap (แผนพัฒนาระบบ)

แสดง Timeline ของการพัฒนา PEA Brain แบ่งเป็น Phase:

| Phase | ช่วงเวลา | เนื้อหา |
|---|---|---|

## 10. คำถามที่พบบ่อย (FAQ)

### Q1: ข้อมูลในระบบอัปเดตบ่อยแค่ไหน?
**A:** ข้อมูลโหลดจาก API ทุกครั้งที่เปิดหน้าระบบ สามารถกด Refresh (F5) เพื่ออัปเดตใหม่

### Q2: AI ใช้ข้อมูลอะไรในการวิเคราะห์?
**A:** AI ใช้ข้อมูลจริงจาก:
- สต็อกปัจจุบัน + Safety Stock
- ประวัติเบิกจ่ายย้อนหลัง 12 เดือน
- Lead Time จัดส่ง
- ราคาต่อหน่วย + ราคากลาง
- EOQ (Economic Order Quantity) ที่คำนวณจากสูตร
- ข้อมูลจาก RAG Knowledge Base ของ PEA

### Q3: แผนที่อนุมัติแล้วเก็บไว้ที่ไหน?
**A:** เก็บใน localStorage ของ Browser แต่ละเครื่อง ไม่หายเมื่อปิด Browser  
> ⚠️ จะหายถ้า Clear Browser Data

### Q4: ทำไม Plan B ไม่มีข้อมูล?
**A:** ถ้า AI ส่ง JSON กลับมาไม่ครบ ระบบจะใช้ข้อมูลจริงจากพัสดุ (สต็อก, ราคา, Demand) เติมให้อัตโนมัติ ไม่มีช่องว่าง "-" อีกแล้ว

### Q5: ใช้งานบนมือถือได้ไหม?
**A:** ได้! ระบบเป็น PWA (Progressive Web App) รองรับทุกขนาดหน้าจอ:
- กดปุ่ม ☰ เปิด Sidebar
- แชท AI เปิดเป็น Fullscreen บนมือถือ
- ตารางเลื่อนซ้าย-ขวาได้

### Q6: "ดาวน์โหลด Excel" อยู่ที่ไหน?
**A:** อยู่ในหน้า AI วิเคราะห์ (E-Bidding) ส่วน "ตารางแผนจัดซื้อ" จะมีปุ่ม 📥 Download Excel

### Q7: สามารถลบแผนที่อนุมัติไปแล้วได้ไหม?
**A:** ในเวอร์ชันปัจจุบัน ไม่สามารถลบแผนที่อนุมัติแล้วได้โดยตรง แต่สามารถ "อัปเดตแผนใหม่" (กดเลือกแผนซ้ำ) ซึ่งจะเพิ่มแผนใหม่เข้าไป
